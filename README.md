## Projects

### Low-Latency Systems

**[Ultra Low-Latency Order Book Engine](https://github.com/Tirth0019)** `C++17`  
Exchange-style L2 order book with strict price–time priority. Cache-conscious memory layout, pre-allocated object pools, lock-free coordination. Built to understand what latency actually costs at the hardware level.

### Quantitative Finance

**[Multi-Timeframe Algo Trading Bot](https://github.com/Tirth0019)** `Python`  
Multi-timeframe decision pipeline with market structure detection (BOS/CHOCH), volatility-normalized trend scoring, and a custom candle-by-candle backtest engine. Reduced false signals ~30–40% via sequential filtering.

**[Bayesian Portfolio Optimization](https://github.com/Tirth0019)** `Python · PyMC · CVXPY`  
Replaced classical Markowitz with Bayesian inference over expected returns + Hidden Markov Model regime detection. Uncertainty-aware allocation, optimized under real-world constraints with CVXPY.

### Full-Stack & Systems

**[Kovari](https://github.com/Tirth0019)** `Next.js · TypeScript · Supabase · Redis`  
Solo travel matching platform. ML-based compatibility ranking across destinations, dates, budgets, and interests. Production Redis with TTL-enforced session management. Built and led as founder.

**[FinIdeas — Lead Management PWA](https://github.com/Tirth0019)** `Next.js · Spring Boot · PostgreSQL` · *HTT 2026*  
Offline-first PWA for event lead capture. My contribution: full frontend, admin dashboard with analytics, and OCR visiting-card scanner (Tesseract.js). Syncs to backend when connectivity returns.

**[Distributed Ride Booking System](https://github.com/Tirth0019)** `Node.js · Redis · Docker`  
5-service microservices system simulating Uber-scale ride matching. My contribution: Driver Service — atomic locking via Redis `SET NX` with TTL to prevent double-booking under concurrent load.

**[CodeReviewEnv](https://github.com/Tirth0019)** `Python · FastAPI · Docker` · *Meta × Scaler*  
OpenEnv-compliant RL environment for evaluating LLM code review agents. Three task tiers (easy → hard): bug detection, security review, concurrency analysis. Baseline: 0.82 avg score on llama-3.3-70b.
